首次使用 · 环境配置
1
确认 Python 版本
需要 Python 3.8 或以上版本。
python --version
2
安装依赖库
在脚本同级目录下打开终端运行：
pip install pandas openpyxl playwright playwright-stealth
3
安装浏览器内核
必须执行，下载 Playwright 专属 Chromium：
playwright install chromium
✅

环境只需配置一次，后续直接点击上方按钮或双击 run.bat 运行即可。

🚀
日常使用 · 运行步骤
1
准备 ASIN 输入文件
点击上方「ASIN 输入列表」打开 asin_list.xlsx，填写 Site 列和 ASIN 列，可直接追加新行。
2
选择模式启动
点击上方「模式 1」或「模式 2」按钮，弹出 CMD 窗口自动运行。
3
人机协同过验证码
遇到验证码时控制台发出警报并暂停，在弹出的 Chrome 窗口中手动滑过，程序自动恢复。
4
查看结果
运行完毕后点击上方「图片链接结果表」直接打开 amazon_images_result.xlsx。
🖥️
两种运行模式说明
1
接管日常 Chrome
强杀后台残留进程，复用日常 Cookies，亚马逊验证码出现率极低。需要关闭当前所有 Chrome 窗口。
推荐
2
独立调试模式
无需关闭正在使用的 Chrome，开启隔离 Profile 互不影响。首次运行需手动登录一次亚马逊。
备用
⚠️

选择模式 1 前请确保已保存所有 Chrome 标签页内容，脚本会强制关闭所有 Chrome 进程。

增量追加去重机制
① 启动时自动读取历史
脚本启动时自动读取现有的 amazon_images_result.xlsx，提取所有状态为 Success 的 ASIN 任务。
② 自动跳过已成功项
主循环中自动跳过已成功的 ASIN，防止重复请求，保护账号安全并大幅缩短运行时间。
③ 合并去重写回
本次新结果与历史数据在 [Site, ASIN] 维度合并去重，以最新状态为准，历史数据不丢失不覆盖。
🛠️
常见异常处理
异常现象根本原因解决方案
ModuleNotFoundError 尚未安装某 Python 第三方包 重新执行 pip install 安装对应依赖
Playwright initialization error 未安装 Playwright 专用浏览器内核 在终端执行 playwright install chromium
所有 ASIN 显示 Dog / Unavailable 本地 IP 被封锁或验证码未成功通过 重新运行并及时在弹窗中滑过验证码；大批量建议分批次运行
按钮显示灰色无法点击 Launcher Service 未运行 通过 启动Hub.bat 启动 Hub，Launcher 会自动随之启动
ECONNREFUSED ::1:9222 Windows IPv6 解析偏好冲突 脚本已内置自愈机制，自动替换为纯 IP 127.0.0.1 连接，无需手动处理
