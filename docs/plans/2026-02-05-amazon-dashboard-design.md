# 亚马逊运营周复盘看板 (Amazon Weekly Review Dashboard) 设计方案

## 1. 项目概述 (Overview)
打造一个基于 Web (Python Streamlit) 的本地化数据看板，整合销售、广告、库存全链路数据。解决数据分散在多张 Excel 表格（领星、简道云、Amazon 后台、手工表）导致复盘效率低下的问题。

## 2. 核心功能 (Core Features)

### 2.1 数据摄入与清洗 (Data Ingestion)
- **拖拽上传**: 支持一次性拖入多个 Excel/CSV 文件。
- **智能识别**: 
  - 自动识别领星导出的库存表、销售表。
  - 自动识别亚马逊/领星导出的广告报表。
  - 自动识别简道云导出表 (工厂端) 和 物流船期表。
- **数据清洗**:
  - **库存表解析**: 重点提取 FBA库存、国内仓库存。
  - **在途解析**: 整合船期表与物流在途数据。
  - **广告表解析**: 提取 `搜索词`、`广告活动` 等核心维度。

### 2.2 库存全链路推演 (Inventory Simulation)
- **全链路视图** (简化版): 对每个 SKU 建立时间轴：
  - `FBA 现货` -> `海上/空中在途` -> `国内仓` -> `工厂在产`。
- **断货预警**: 
  - 结合 `7天/30天日均销量` 预测 FBA 断货时间点。
  - 标记 **红色预警**: FBA 可售天数 < 15天。
  - 标记 **黄色预警**: 海上在途 + FBA 总量无法支撑接下来的旺季。

### 2.3 广告与销售复盘 (Ads & Sales Review)
- **ASIN 诊断**: 点击任意 ASIN，展示其：
  - **流量漏斗**: 曝光 -> 点击 -> 转化。
  - **关键词表现**: 自动列出花钱最多但转化差的 "吸血词" (High Spend, Low ROAS)。
  - **排名监控**: (如有数据) 展示关键词排名趋势。

### 2.4 异常监控中心 (Anomaly Detection)
- **自动生成异常清单**:
  - **滞销**: 库龄 > 90天 且 动销率极低。
  - **广告异常**: 单次点击成本 (CPC) 飙升 > 30% 或 ACOS > 利润率。
  - **退货异常**: 退货率突然升高的 SKU。

## 3. 技术架构 (Technical Architecture)
- **语言**: Python 3.10+
- **前端框架**: Streamlit (快速构建美观的 Web UI，符合 `ui-ux-pro-max` 审美)。
- **数据处理**: Pandas (作为 ETL 引擎)。
- **图表库**: Plotly (交互式图表) / AgGrid (高性能交互表格)。
- **部署**: 本地运行 (`d:\Zero Tools\dashboard.py`)，无需服务器，数据安全。

1. **MVP 阶段 1**: 编写 `Parsers` 模块，专门解析刚才分析过的两个 Excel 模板，确保能正确读取数据。
2. **MVP 阶段 2**: 搭建 Streamlit 框架，实现文件上传和基础表格展示。
3. **MVP 阶段 3**: 实现“库存推演”逻辑和“红绿灯”预警界面。
